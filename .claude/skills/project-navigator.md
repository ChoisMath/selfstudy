# 프로젝트 네비게이터

## 목적

새 세션에서 전체 소스코드를 읽지 않고 `.claude/PROJECT_MAP.md`를 참조하여 코드베이스를 빠르게 파악하고 작업을 시작합니다. 코드 변경 시 지도를 자동 업데이트합니다.

## 사용 시점

### 1. 세션 시작 시 (필수)
새 세션이 시작되면 가장 먼저 `.claude/PROJECT_MAP.md`를 읽어 프로젝트 구조를 파악합니다.

### 2. 코드 변경 후 (자동)
파일을 생성, 수정, 삭제한 후 해당 변경이 지도에 영향을 미치는 경우 지도를 업데이트합니다.

## 세션 시작 워크플로우

1. **Read** `.claude/PROJECT_MAP.md`
2. 지도의 "마지막 업데이트" 날짜를 확인하여 최신 여부 판단
3. 사용자의 작업 요청과 관련된 섹션을 참고하여 필요한 파일만 읽기
4. 작업 수행

## 코드 탐색 가이드

지도에서 작업 관련 영역을 찾는 방법:

| 작업 유형 | 지도 참조 섹션 | 읽을 파일 |
|----------|--------------|----------|
| API 수정/추가 | "API 라우트 요약" | 해당 route.ts + lib/api-auth.ts |
| UI 변경 | "디렉토리 구조" | 해당 page.tsx + 관련 컴포넌트 |
| 인증/권한 | "인증/인가 흐름" | lib/auth.ts, middleware.ts, lib/api-auth.ts |
| DB 스키마 | "데이터 모델" | prisma/schema.prisma |
| 비즈니스 로직 | "핵심 비즈니스 로직" | 해당 API route.ts |
| 좌석 관련 | "실제 교실 구조" + 컴포넌트 | components/seats/* |
| 배포 이슈 | "배포 정보" | package.json, .env |

## 지도 업데이트 트리거

다음 변경 시 `.claude/PROJECT_MAP.md`를 업데이트합니다:

- **파일 생성/삭제**: 디렉토리 구조 섹션 업데이트
- **새 API 라우트**: API 라우트 요약 섹션에 추가
- **Prisma 스키마 변경**: 데이터 모델 섹션 업데이트
- **새 컴포넌트**: 디렉토리 구조 + 해당 설명 추가
- **인증/권한 변경**: 인증/인가 흐름 섹션 업데이트
- **배포 설정 변경**: 배포 정보 섹션 업데이트
- **비즈니스 로직 변경**: 핵심 비즈니스 로직 섹션 업데이트

## 업데이트 방법

1. 변경된 파일을 확인
2. `.claude/PROJECT_MAP.md`에서 관련 섹션을 찾아 Edit으로 수정
3. "마지막 업데이트" 날짜를 오늘로 변경

## 주의사항

- 지도는 **구조와 흐름**을 기록하는 것이지, 코드 자체를 복사하는 것이 아닙니다
- 실제 구현 세부사항이 필요하면 해당 파일을 직접 Read하세요
- 지도와 실제 코드가 충돌하면 **실제 코드가 우선**입니다 — 지도를 수정하세요
